What is source/version control?

A component of software configuration management, version control, also known as revision control or source control, is the management of changes to documents, computer programs, large web sites, and other collections of information.

Why do I care?

  • Allows you to ensure that code is in a working state
  • Prevents a single computer from being the be all end all for housing in-progress code
  • Make collaboration easier and attempts to prevent merge conflicts

Examples of Source Control

  • Microsoft Office Track Changes
  • Google Docs History and Suggestions
  • Other Technical Kinds
    • Mercurial
    • Subversion

What is Github?

Git and Github are NOT the same!

Git is the version control system.

Github is the user interface that utilizes git. You can use git without Github.

Vocabulary

  • Local: The machine you're working on
  • Remote: The server where the code lives that is not your computer
  • Repository (Repo): Folder/Directory
  • Commit: Defined version of your code and the changes you are making
  • Pushing: Moving code from your machine to the server
  • Pulling: Bringing changes down to your machine from the server
  • If you're using Github for Desktop, "Sync" Pulls and Pushes to the remote

Collaboration in the Real World: Keeping Your Stuff Separate

  • Branching: An offshoot of the main code base
  • Forking: Full Clone of a repository, completely separate from the code base

Collaboration in the Real World: Bringing It All Together

  • Cloning and Merging or Rebasing
    • Everyone has full access
    • Can cause a lot of conflicts that have to be resolved
  • Pull Request
    • Easier to see the files that everyone changed
    • Adds an extra step
    • Can be assigned to others for code review

Rebasing versus Merging

  • Rebasing: Rewrites the history of the branch/commits. This usually means that it puts commits after all other completed work on the existing branch.
  • Merging: Puts commits back into the history based on where the branches diverged. Commits are put in chronological order.
